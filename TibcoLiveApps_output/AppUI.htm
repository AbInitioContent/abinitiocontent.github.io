TIBCO Cloud Live Apps

The following screen grabs are taken from TIBCO Cloud Live Apps - a cloud based BPM solution. The aim was to take a potentially complex application and make it possible for users to create usable apps immediately. Within the Agile framework, I worked with UX designers and developers to ensure that app creation was intuitive, and that users did not have to leave the UI for assistance. Iterative design, with user testing, supported this approach.

Assistance front and center on first visit

If they choose to, the user can take one or more forms of assistance to get them started when they first use Live Apps.

Guided through their first application

If they choose to, the user can create their first application with the help pf a dialog walk-through.

The user follows instruction on the interactive dialog.

...through a series of steps...

The dialog is contextual depending on the options selected by the user.

Helping users understand what they've created

To provide context and understanding on their first application, the user can watch a video that describes what they've built in detail, including how they can configure other options and test the application. This is offered immediately after build.

Helping users understand the key aspects of the interface

Interactive tours provide context and understanding.

Assistance is always available when creating new apps

Users can build new apps by using a wizard that helps them with each step. More confident users can choose to create an application without assistance by building from a template.

Convenient means of configuration

Clicking or tapping configurable options opens a slider dialog on the right of the screen.

Instruction to complete the configuration are clearly defined in the slider panel and users are guided by simple questioning.

Never miss the detail

The user does not need to worry about things they might forget as they build their app. The interface has a 'to do' list, helping users complete the necessary configuration options.

Oops, something went wrong

Messages are clearly displayed with recommended actions

Help is always available

Individual screens and interface items have their own contextual help resource. No need for the user to leave the interface.
