TIBCO Cloud Live Apps

The following screen grabs are taken from TIBCO Cloud Live Apps - a cloud based BPM solution.
We wanted users to be able to create BPM flows that they could test within minutes. The interface needed to be intuitive and conversational. If the user got stuck or needed help, resources needed to be available within the interface and within context.
Within the Agile framework, I worked with UX designers and developers to support this ideal. Iterative design, with user testing, underpinned the approach.

Assistance front and center on first visit

If they choose to, the user can take one or more forms of assistance to get started with Live Apps.

Guided through their first application

The user can create their first application with the help of a dialog walk-through.

Task instructions are provided within logical steps.

The dialog is contextual depending on the options selected by the user.

Helping users understand what they've created

To provide context and understanding on their first application, the user can watch a video that describes what they've built in detail, including how they can further configure and test their application. The video is offered immediately after their first build.

Assistance is always available when creating new apps

Users can build new apps by using a wizard that helps them with each step. More confident users can choose to create an application without assistance by building from a template.

Helping users understand the key aspects of the interface

Interactive tours provide context and understanding.

Convenient means of configuration

Clicking or tapping configurable options opens a slider dialog on the right of the screen.

Instruction to complete the configuration are clearly defined in the slider panel and users are guided by conversation and simple questioning.

Never miss the detail

The user does not need to worry about things they might forget as they build their app. The interface has a 'to do' list, helping users complete the necessary configuration options.

Clear messaging

Messages are clearly displayed with recommended actions.

Help is always available

No need for the user to leave the interface for assistance. Individual screens and interface items have their own contextual help resource.
