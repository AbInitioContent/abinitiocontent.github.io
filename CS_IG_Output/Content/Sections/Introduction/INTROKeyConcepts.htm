Key concepts

The IG Server introduces a number of concepts:

Roles

An IG Server user can have one of three roles:

  • Compliance Officer
  • Document Owner
  • Administrator

The role of a particular user determines which parts of the user interface the user can access, and which tasks they can carry out. There can be more than one user in each role.

You must have at least one Compliance Officer set up. Other roles are optional, and depend on whether a Compliance Officer wants to delegate responsibility for tasks. A Compliance Officer is effectively a “super-user” who has full access to the system and can carry out all tasks.

Show me the roles and what they can do:
Role Number allowed Description
Compliance Officer 1 or more A “super-user” that can access all parts of the user interface and carry out any task.
Document Owner 0 or more A user that can register items and manage collections of documents.
Administrator 0 or more A user that has Document Owner privileges. In addition, is responsible for controlling who has access to the system, which Gateways connect to the system, and ensuring the system remains up-to-date and operational.

Collections

The purpose of a collection is to:

  • Organise registered items into logical groups
  • Define the scope of whitelisted text
  • Provide context to people dealing with policy violations

When you first register an item, you are required to specify a collection to add it to. Every item is associated with exactly one collection, although items can be moved between collections.

Collection owner

Each collection needs a collection owner. Collection owners can be a Compliance Officer, Administrator, or a Document Owner.

When a user creates a new collection, they become the initial collection owner.

Each collection has a unique and permanent ID which is automatically generated by the IG Server. A display name is supplied which can be re-configured by the collection owner.

Collection owners:

  • receive email notifications in the event of a match against items in the collection
  • are responsible for dealing with false-positives by whitelisting text or deregistering items that should no longer be registered
  • can register and deregister items in the collection
  • can see which items in the collection have caused most matches
  • can modify details on collections that they own
  • can delete collections that they own

Item registration

You register items that you want to be inspected for matches as part of your organization's security policy. When you register an item, the IG Server generates full file hashes and fingerprints for the registered item. Content that passes through your Gateway can then be checked for matches against stored hashes and fingerprints, and any violations can be handled accordingly.

Note:  When registering encrypted files, only a full file hash is generated. This means that partial matching only can be performed on the item.

Item classification

When registering items on the IG Server, document owners will be able to select an optional classification that allows for extra flexibility when defining Gateway policy. The classifications are based around the level of document sensitivity with default definitions as follows:

  • Top Secret
  • Secret
  • Confidential
  • Restricted
  • Unclassified
  • Unspecified

The item classification is used when responding to Gateway queries. The IG Server reports the highest classification of all the matching items. It also returns the number of matching items and matching fingerprints at each level.

Item properties

When you register a new item on the IG Server a number of properties need to be supplied:

Property Description Registration
User Identity of the user registering the item Derived automatically by the IG Server
User's IP address IP address of user’s PC at the time of registration Derived automatically by the IG Server
Timestamp Time of registration Derived automatically by the IG Server
Filename The name of the file that was registered Taken from the uploaded file
Location The location of the file that was registered Not specified
Expiry The date the item will be automatically deregistered Supplied by the user (default is never)
Collection name The name of the target collection Supplied by the user
Custom Properties See note Supplied by the user (optional)
Custom Properties

These are name and value pairs supplied at registration time that provide extra contextual information for each registered item. In addition, your organization's policy might make use of these properties in order to define different actions depending on the properties of the matching items.

Custom properties have no special meaning to the IG Server; it simply stores them for each item and returns them in query results for matching items. For example, the properties might indicate a particular functional department, for example, Finance. You enter name and value pairs as free format text.

Whitelisting

To lessen the chances of false positives where items are in a collection are matched to content found in an information flow, you can add a fragment of text or a whole document to a whitelist. Items in a whitelist are not used for matching purposes. Whitelist items would typically include document templates and boilerplate text.

Unless you use common whitelisting, whitelist items are specific to a particular collection; they only apply to items that are part of that collection.

Common Whitelisting

Any file or text that is added as a common whitelisted item will be applied across all collections. It therefore provides a convenient method of adding common content, for example company logo's and copyright statements, across all collections without the need to register it individually across multiple collections.

Common whitelisting is available to Compliance Officers only.

Tip:  We recommend that Compliance Officers (CO) create a new collection specifically to contain common whitelist items. In addition, the items should not be moved into collections not owned by the CO or any other CO, nor should the CO change the owner of a collection containing common whitelist items to a user that is not a CO.
