Overview

The Clearswift Information Governance Server (IG Server) enables your organisation to identify important information, discover the whereabouts of similar information, and monitor and control the flow of this information with the organisational boundary and beyond.

The IG Server, along with other Clearswift products, provides the basis for the Clearswift Information Governance Platform (IG platform). Collectively, the IG platform allows you to register, control, and track content. The IG platform comprises the following:

  • The Information Governance Server
  • One or more Clearswift SECURE (Web, Email, Exchange, or ICAP) Gateways

Tell me about...

  • The role of the IG Server within the IG platform

    The IG Server is the primary mechanism for content registration, and it acts as the repository for your important information. It also hosts tracking information and can be used to report on the movement of registered data.

    In addition, the IG Server provides notification emails to alert IG Server users where the Gateway has detected a match with registered items.

  • The role of the Clearswift Gateways within the IG platform

    The Gateways provide the content control within the platform by means of policy rules that compare IG registered content against web transactions and email content that flows through the Gateway itself.

    The SECURE Email Gateway (SEG), and SECURE Web Gateway (SWG), and SECURE Exchange Gateway (SXG) interact with the IG Server in two ways:

    • They query the Content Registry to determine whether emails or web transactions include content that matches any registered content so that they can apply policy accordingly
    • They submit information regarding email attachments and web uploads and downloads to the tracking server for content tracking purposes

    The SECURE Exchange Gateway (SXG) is a variant of the SEG designed to process internal email by integrating with Microsoft Exchange Server. The SXG automatically inherits the IG Server features of the SEG

  • The level of matches that are supported

    The IG registered content is compared against web transactions and email content by using policy rules provided by the Gateways. The following types of matches are supported:

    • Email attachments that are identical to any registered item
    • Web uploads (files) that are identical to any registered item
    • Web downloads (files, not parts of web pages) that are identical to any registered item
    • Email attachments containing text that partially matches any registered document
    • Email bodies that partially match any registered document
    • Web uploads (files) containing text that partially matches any registered document
    • Web downloads (files, not parts of web pages) containing text that partially matches any registered document
