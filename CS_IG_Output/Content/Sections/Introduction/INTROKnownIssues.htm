Known issues in IG Server 1.0

The following are known issues (with workarounds where possible) in this version of the IG Server:

  • Unable to access the IG Server administration user interface if the account for the directory server expires or there is a password change

    You have to supply directory server account credentials as part of your IG Server configuration. It is possible to lock all users out of the IG Server administration user interface if the account expires or there is a password change.

    What can I do?

    You should create a service account for this purpose and set password expiry to disabled.

  • Registering whitelist items when part of an archive file only displays the registration result for the first item

    If you register whitelist items using an archive file (zip, tar etc), the Administration UI will only display the registration result for the first item in the archive. Therefore, you might encounter the following situations:

    • If first item registration is successful, the UI will indicate that the archive was registered successfully even if subsequent items might have failed registration
    • If first item registration is failure, the UI will indicate that the archive failed to register even if subsequent items might have registered successfully
    What can I do?

    We recommend that you check that all items have been registered successfully on the Registry page after whitelisting the archive. Any items not registered should be extracted from the archive and registered individually to determine the reason for failed registration.

  • Gateway tracking and reporting data restrictions

    Transaction data is sent from your Gateway to your IG Server for the purposes of reporting. Data from the following Gateway transactions are not supported:

    • Released messages from PMM
    • Released messages from Trusted Sender
    • Released messages from Inform
    • Additional recipient on Missing Manager rule
