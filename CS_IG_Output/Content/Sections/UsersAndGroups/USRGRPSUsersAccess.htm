Manage user access

To use the IG server, users must have:

  • A valid Active Directory (AD) username in your IG Server deployment
  • An assigned IG Server role

When AD authentication is successful, the IG Server determines the user's role based on the configured list of users and groups as set up on the IG Server. Within the IG Server, these are known as security principals.

  Any user with a valid AD account in your IG Server deployment can log in to the IG Server with Document Owner permissions by default, without the need to be added as an individual user.

Tell me about...

  • Roles

    An IG Server user can have one of three roles:

    • Compliance Officer
    • Document Owner
    • Administrator

    The role of a particular user determines which parts of the user interface the user can access, and which tasks they can carry out. There can be more than one user in each role.

    You must have at least one Compliance Officer set up. Other roles are optional, and depend on whether a Compliance Officer wants to delegate responsibility for tasks. A Compliance Officer is effectively a “super-user” who has full access to the system and can carry out all tasks.

    Show me the roles and what they can do:
    Role Number allowed Description
    Compliance Officer 1 or more A “super-user” that can access all parts of the user interface and carry out any task.
    Document Owner 0 or more A user that can register items and manage collections of documents.
    Administrator 0 or more A user that has Document Owner privileges. In addition, is responsible for controlling who has access to the system, which Gateways connect to the system, and ensuring the system remains up-to-date and operational.
  • Groups

    You can add multiple users to a given role by adding a group as defined in Active Directory (AD). Adding a group in this way means that any member of that group can log in to the IG Server in the specified role.

    To avoid groups of users being given Compliance Officer access, a group can be assigned the Document Owner role only. Adding and removing users to AD will affect IG Server login accordingly.

    If a user is assigned the Document Owner role as part of a group addition, a Compliance Officer or IG Administrator role already assigned will take precedence.

How do I...

  • See a list of IG Server users and groups?

    From the toolbar, click Users and select the User access tab.

    The list of users and groups with associated role is displayed.

  • Add a user or group?

    1. From the toolbar, click Users and select the User access tab.

      The list of users and groups with associated role is displayed.

    2. Click .

      The Add user or group dialog is displayed, and lists groups and individual users as configured in Active Directory.

    3. Click the user or group you want to add, and select the role from the Role drop down.

       

      Use the search field to quickly find specific users or groups.

    4. Click OK.

  • Delete a user or group?

    1. From the toolbar, click Users and select the User access tab.

      The list of users and groups with associated role is displayed.

    2. Click against the user or role you want to delete.

    3. Click OK in the confirmation box.

  • Change the role of a user or group?

    1. From the toolbar, click Users and select the User access tab.

      The list of users and groups with associated role is displayed.

    2. In the Role column for the user or group you want to change, click and select the role from the drop down list.

    3. Click to confirm your selection.
