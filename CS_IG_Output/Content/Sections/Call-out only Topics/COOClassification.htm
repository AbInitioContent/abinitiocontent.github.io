Configure Classification

When registering items, IG Server users can apply a classification to indicate the level of document sensitivity. This classification is used when a Gateway queries the IG Server. The IG Server will report the highest classification of all the matching items and will return the number of matching items and fingerprints at each classification level.

This feature can be enabled or disabled. If enabled, users must select a defined classification when registering an item. If disabled, items will be registered with a level of unspecified by default.

Your IG Server is supplied with a default list of classifications. You can use this list as part of your organization's policy or supply your own.

All child items of registered items inherit the parent item classification level.

Tell me about...

  • The Importance, Level, and Description columns on the Classifications configuration pane and how they relate to classification

    The Importance column provides a visual representation of your classification. It indicates that descriptions of your most important classification should be placed at the top of the list and against the highest numerical level.

    The Level is a non-configurable value that is used by your IG platform for querying and reporting. Although the number is internal to the IG Server it is helpful when assigning descriptions, and it is this value that is used by the Gateway for threshold levels when configuring policy.

    The Description is a configurable value that is assigned to a given level. For example, you might want to assign 'Top Secret' to the highest available level of 900. Provided that classification is enabled, the list of descriptions (and levels) are available for selection when items are registered. Levels that have no description applied will not be displayed when registering items.

  • What happens to items already registered with a classification if I change the description

    Changing a classification will not effect items already registered. This is because the item is assigned the level number and this remains unchanged even if the description is changed; it is the interpretation of the level number that changes. For example, an item might be given a classification of Secret (700). If the classification is subsequently changed to Confidential, the level remains at 700.

How do I...

  • Enable Classifications and set level descriptions
    1. In the Classifications panel, click Edit.
    2. Select the Ask user to enter a classification when registering items check box.
    3. Edit the Description fields for the appropriate level as defined by your naming policy.
    4. Click Save.
